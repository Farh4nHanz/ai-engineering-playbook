## ✨ What is this?

A collection of **structured AI workflows** designed to make LLMs behave like senior engineers across the full development lifecycle:

- 📄 PRDs (Product thinking)
- 🏗️ Feature planning & implementation
- 🧪 Testing & QA
- 🔍 Debugging
- 🧹 Refactoring
- 👀 Code review

## 🧠 Available Skills

| Skill                           | Description                                      |
| ------------------------------- | ------------------------------------------------ |
| `product-requirements-document` | Create lightweight but implementation-ready PRDs |
| `feature-delivery-end-to-end`   | Plan and build features step-by-step             |
| `senior-dev-code-review`        | Perform production-grade code reviews            |
| `testing-engineer`              | Design high-coverage test suites                 |
| `clean-code-refactor`           | Refactor messy code using clean code principles  |
| `senior-debugger`               | Debug issues and identify root causes            |

## ⚡ Quick Start

Choose how you want to use the skills based on your LLM:

---

### 🤖 Using Claude (Recommended)

Claude supports structured prompts very well, making it the best experience for this playbook.

1. Download the skill file you want
2. Upload it to Claude
3. Use it by typing `/<skill-name>`

---

### 💬 Using ChatGPT / Other LLMs

1. Use the prompt template:
   👉 `template/prompt-template.md` or see `examples/<skill-name>` and copy the **Input** section

2. Select a skill:

    ```md
    You are using the "senior-debugger" skill.
    ```

3. Provide your input:

    ```md
    Here is my code:
    [PASTE CODE]

    Here is the issue:
    [DESCRIBE BUG]
    ```

## 🧩 How It Works

Each skill:

1. Defines a **clear role** (e.g., senior engineer, QA, tech lead)
2. Enforces a **structured workflow**
3. Produces **consistent, high-quality output**

This removes randomness from AI responses and makes them **predictable and production-ready**.

---

## ⚠️ Important Note

If you provide **too much context**, the AI may skip the clarification phase.

👉 If you want:
**Questions first → then solution**

Keep your input slightly ambiguous.
